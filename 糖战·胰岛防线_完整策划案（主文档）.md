## 一、设计历程

这个游戏的核心灵感，源于一组触目惊心的公共卫生数据——全球糖尿病患病人数已超过 5.37 亿，且呈年轻化趋势持续蔓延。在中国，每 10 个成年人中就有超过 1 人罹患糖尿病，而知晓率和治疗率却远低于应有水平。

作为一名计算机游戏开发的学习者与实践者，我在思考一个根本性的问题：**游戏除了娱乐，还能承载什么？**

本作的初始构思，来自于一次将"人体免疫系统对抗疾病"与"纵版射击游戏"进行类比的思想实验——免疫细胞在血液中巡航、识别并消灭病原体，这与战机在弹幕中穿梭、锁定并摧毁敌人的玩法何其相似！而糖尿病这种以"高血糖慢性侵蚀全身器官"为核心的代谢性疾病，恰好提供了从"血液"到"视网膜"再到"胰腺"的天然关卡递进结构。

在参考了大量关于糖尿病防治的科普资料（包括《中国 2 型糖尿病防治指南》、WHO 糖尿病专题等权威来源）后，我决定将糖尿病相关的病理机制——高血糖、血液黏稠、视网膜病变、胰岛功能衰竭等——逐一转译为游戏中的敌人、Boss、地图与环境危害，让玩家在"打飞机"的过程中，不知不觉地理解糖尿病的危害与预防知识。

在设计方法论上，本作参考了经典纵版射击游戏"雷霆战机"的核心操作循环（移动 + 自动开火 + 大招），借鉴了"生化危机"系列对生物恐怖的氛围营造手法，以及《靛青预言》（Indigo Prophecy）对"游戏即叙事"的设计哲学。但本作的核心差异化不在于技术或玩法创新，而在于**题材即玩法**的原创思路——游戏的每一个敌人、每一张地图、每一次死亡，都在讲述一个关于糖尿病的真实故事。

> 如果说"雷霆战机"让玩家在太空中成为英雄，那么"糖战·胰岛防线"让玩家在人体内成为救世主——而这个人体的主人，可能就是我们身边的每一个人，甚至是我们自己。

## 三、游戏概述

### 3.1 一句话简介

> 你是注入患者体内的纳米战机驾驶员，在血管、器官与神经的黑暗深处，击退因糖尿病而失控、异变、增殖的"高糖怪物"，拯救一具濒临崩溃的身体。

### 3.2 设计目标

| 维度 | 目标 |
| --- | --- |
| **可玩性** | 上手即玩的纵版射击，操作 ≤ 3 个按键 |
| **科普性** | 每关、每个 Boss、每次死亡都自然嵌入糖尿病知识 |
| **警示性** | 用"体内崩坏"的恐怖氛围，让玩家直观感受高血糖的危害 |
| **可完成性** | 2 周内由小团队（1-3 人）做出完整可玩版 |

### 3.3 核心卖点

- **科幻恐怖的人体内景**：在搏动的血管、溃烂的视网膜、坏死的神经中战斗，画面压抑、惊悚。
- **题材即玩法**：敌人、地图、Boss 全部由真实糖尿病病理转译而来，玩法本身就是科普。
- **知识闭环**：战斗 → 死亡 / 通关 → 弹出"病理档案"卡片 → 玩家记住一个知识点。

### 3.4 与"雷霆战机"的异同

| 维度 | 雷霆战机 | 本作《糖战·胰岛防线》 |
| --- | --- | --- |
| 视角 | 纵版卷轴 | 纵版卷轴（一致） |
| 操作 | 移动 + 自动开火 + 必杀 | 移动 + 自动开火 + 大招（胰岛素脉冲） |
| 主题 | 太空科幻 | 人体微观 / 糖尿病病理 |
| 场景 | 星际、基地 | 血管、视网膜、胰岛 |
| 敌人 | 外星战机、机械 | 糖晶虫、变异白细胞、高糖孢子等（全部由病理转译） |
| 附加目标 | 纯娱乐 | 娱乐 + 科普 + 健康警示 |
| 美术风格 | 科幻金属、爆炸 | 生物科幻恐怖、黏腻有机质感 |
| 死亡机制 | 简单续命 / Game Over | 死亡读条期间弹出糖尿病冷知识 |
| 通关奖励 | 得分排行 | 得分 + 病理档案卡片 + 图鉴解锁 |
| 道具隐喻 | 能量、炸弹 | 每件道具对应真实控糖手段（饮食/运动/胰岛素） |
| 知识闭环 | 无 | 战斗→死亡/通关→科普卡片→记住知识点 |

### 3.5 哲学观点

**哲学观点 #1 —— 游戏能否成为健康教育的载体？**

传统的健康教育手段——宣传册、课堂讲座、公益广告——在信息过载的时代越来越难以抓住受众的注意力。而游戏作为一种"强迫性互动"的媒介，天然具有让人"亲身经历"而非"被动接受"的优势。在本作中，玩家不是被告知"高血糖有害"，而是**亲眼看到**被糖化结晶覆盖的血管内壁，**亲手对抗**由血糖失控凝聚的怪物，**亲自尝到**"高糖陷阱"（伪装成甜甜圈的负面道具）带来的后果。这种沉浸式的"体验式学习"，比任何说教都更有说服力。

**哲学观点 #2 —— 恐怖美学服务于警示目的**

为什么选择"恐怖"作为本作的美术基调？因为糖尿病的真实病理过程本身就是一场"体内的恐怖片"——微血管破裂出血、视网膜病变致盲、神经坏死溃烂、酮症酸中毒昏迷……这些真实的医学事实，用明亮可爱的卡通风格去表现，不仅不协调，更是一种误导。本作选择用"异形""死亡空间"式的生物恐怖美学来呈现这些病理现象，并非为了惊吓而惊吓，而是为了让玩家**对疾病的可怕产生直观的、情感的、记忆深刻的认知**——从而在现实生活中更重视预防。

**哲学观点 #3 —— 玩法即知识，战斗即科普**

本作的核心设计哲学可以概括为五个字：**"玩法即知识"**。游戏中的每一个元素都对应着真实的医学概念：

- 玩家战机 = 外源胰岛素（治疗手段）
- 高糖怪物 = 失控的血糖
- 胰岛素脉冲大招 = 注射胰岛素降糖
- 膳食纤维道具 = 饮食控制
- 运动能量道具 = 运动降糖
- 高糖陷阱 = 不健康饮食的危害
- 三大关卡 = 糖尿病并发症的演进路径

玩家不需要阅读长篇文字来获取知识——知识就嵌在操作与战斗之中。当你被"高糖孢子"增殖淹没了屏幕，你自然就懂了"拖延治疗的后果"。

### 3.6 基本问题（7 问）

#### 这个游戏是什么？

《糖战·胰岛防线》是一款以糖尿病科普为核心题材的**纵版卷轴飞行射击游戏（STG）**。玩家操控注入患者体内的纳米战机 INS-09，穿越三大人体器官战区——血管（血液战区）、眼底（视网膜战区）、胰腺（胰岛战区），击退因高血糖而失控异变的生物怪物，最终击败终极 Boss"胰岛吞噬者"，重启胰岛素分泌，拯救患者的生命。本作在经典"雷霆战机"式玩法的基础上，将太空科幻替换为人体微观世界的生物科幻恐怖，将外星敌人替换为由真实糖尿病病理转译而来的怪物，并首创"科普闭环"系统。

#### 为什么做这个游戏？

原因有三：

1. **社会需求**：糖尿病是全球最严峻的慢性病之一，中国的糖尿病知晓率、治疗率和控制率均偏低。面向青少年与成人的趣味化健康教育工具极度匮乏——游戏可以填补这一空白。
2. **市场空白**：市面上有大量医疗健康类 APP 和严肃游戏，但几乎没有一款将"硬核射击游戏"与"系统化疾病科普"结合的娱乐产品。"糖战"可以同时吸引射击游戏玩家和健康教育受众，具备跨界传播的潜力。
3. **创作冲动**：将真实的生物学/医学知识融入游戏设计，是极其罕见的创作挑战，也是极具吸引力的创意实验。本作的每一个设计决策——从敌人的行为模式到 Boss 的阶段机制——都必须同时满足"好玩"和"准确"两个标准，这种双重约束反而激发了独特的设计灵感。

#### 这个游戏在哪里发生？

游戏发生在**一位 47 岁糖尿病酮症酸中毒昏迷患者"林先生"的体内**。整个游戏世界是人类身体的微观尺度呈现——血管是幽闭的暗红隧道，视网膜是布满神经网的墨黑世界，胰腺深处是半坏死、濒临停产的胰岛细胞工厂。这个"体内世界"并非健康、温暖、宁静的——它正在从内部腐坏：血管壁挂满糖化结晶痂，血液黏稠如淤泥，微血管破裂喷涌血雾，免疫细胞异变成攻击一切的怪物。玩家所处的，是一个**正在经历"体内末日"的人类身体**。

#### 我要操作什么？

玩家操控的是**纳米战机 INS-09**（胰岛素-09），一艘由医生注入患者体内的医疗纳米机器人飞行器。操作方案极为简洁：

| 操作 | 键盘 | 手柄 | 触屏（预留） |
| --- | --- | --- | --- |
| 移动 | WASD / 方向键 | 左摇杆 | 拖拽战机 |
| 开火 | 自动开火 | 自动 | 自动 |
| 大招"胰岛素脉冲" | Shift / J | LB | 屏幕右侧按钮 |
| 暂停 | Esc | Start | 暂停按钮 |

> **设计原则**：核心仅"移动 + 大招"两件事，符合雷霆战机式的低门槛。开火默认自动。任何玩家在 30 秒内即可上手。

#### 我有几个角色可以选择？

本作仅设**一架可操作战机**：INS-09（胰岛素-09）。这是由设计约束与主题一致性共同决定的选择：

- 从剧情角度看，INS-09 是"唯一的希望"——林先生的病情危急，医生只能注入一架原型纳米战机。
- 从开发周期看，2 周（10 个工作日）内由 1-3 人小团队完成，单一战机大幅降低美术与程序工作量。
- 从科普角度看，单一角色让"INS-09 = 外源胰岛素"的隐喻更加清晰、集中。

虽然只有一架战机，但武器系统提供 **Lv1-Lv5 五级火力升级**，弹道由单发→双发→三向→五向散射→追踪弹逐步强化，赋予玩家充分的成长感。（详见 [第十三章 武器系统与数值设计](#十三武器系统与数值设计)）

#### 主要目的是什么？

玩家的首要目标是**通关三大关卡，击败终极 Boss"胰岛吞噬者"，重启胰岛细胞分泌胰岛素，将林先生从死亡线上拉回来**。

在玩法层面，这意味着：在弹幕地狱中生存、击杀敌人积累能量、收集道具强化火力、击败每个关卡的 Boss、解锁所有科普档案。

在更深的层面，玩家的目标是**通过游戏体验理解糖尿病的危害与预防**。这是一个"拯救"的故事——不仅是拯救游戏中的林先生，也是通过科普警示，让屏幕前的玩家意识到健康的重要性。

#### 与同类游戏有什么不同？

**核心差异化一句话**：本作是唯一一款**"敌人 = 疾病、地图 = 器官、道具 = 药物/生活方式、通关 = 治愈"**的射击游戏。

（详细对比见 [3.4 与"雷霆战机"的异同](#34-与雷霆战机的异同) — 10 维度完整对比表）

## 四、故事背景与世界观

> **设计要求落实**：故事完整、可简短，但必须存在。以下为完整可用的背景设定。

### 4.1 世界观

公元 2045 年，纳米医疗已成熟。当人体陷入危急，医生会注入一支**纳米战机中队**进入患者体内，在微观尺度上直接清除病灶。

这套系统被称为 **G.U.A.R.D.（Glucose Unit for Auto-Regulation & Defense，血糖自律防御单元）**。

这是一场**肉眼看不见的战争**。每一秒，在你不知道的体内深处，数以百万计的细胞正在因高血糖而死去。

### 4.2 主线剧情

患者 **林先生，47 岁，男性**——常年高糖饮食、久坐办公、体重超标，长期忽视身体发出的警报：口渴、多尿、视力模糊、伤口愈合缓慢。他以为"只是累了的正常表现"。

某夜，他被发现昏迷在家中，送医后确诊为**糖尿病酮症酸中毒（DKA，糖尿病最危险的急性并发症之一）**——血糖值飙至超过 30 mmol/L（正常 <6.1），血液 pH 值持续下降，身体正在从内部自我腐蚀。

常规治疗——静脉输注胰岛素、大量补液——启动需要时间。而林先生已经没有那么多时间了。

主治医生紧急启动 **G.U.A.R.D. 应急协议**，将唯一的原型纳米战机 **INS-09** 注入林先生体内。INS-09 搭载 AI 导航系统 **"瑞吉娜"（REGINA）**，任务是：

> 穿越血液循环系统 → 清除高血糖病灶 → 抵达胰腺 → 重启仅存的胰岛细胞 → 把林先生从死亡线上拉回来。

当 INS-09 进入血管的那一刻，玩家看到的不是健康的红色血液——

- 血管壁挂满黄白色的**糖化结晶痂（AGEs 堆积）**；
- 本该是守护者的免疫细胞**异变成攻击一切的高糖怪物**；
- 胰岛——身体最后一座"血糖调控工厂"——已被高糖怪物重重包围，濒临停机。

玩家的任务：**穿越三大器官战区，摧毁失控的高糖病灶，护送最后的胰岛细胞重启分泌，把林先生从死亡线上拉回来。**

这是一场深入人体最黑暗角落的旅程。而成功或失败，只有一个屏幕之隔。

### 4.4 双结局设计（警示设计）

**真结局（通关——胰岛重启成功）**：

屏幕从体内微观世界中淡出，过渡到一间黎明的病房。林先生缓缓睁开眼睛——心电监护仪显示稳定的窦性心率。瑞吉娜 AI 最后一段旁白以平静的语气念出：

> "这次我们赢了。但 INS-09 不可能每天都来。真正的防线——是你每一次放下含糖饮料的选择，是每一次走楼梯而不是坐电梯的决定，是每年体检时那张小小的血糖化验单。真正的防线，是你自己的生活方式。"

屏幕浮现真实的糖尿病预防要点。

**坏结局（耗尽生命 / 失败）**：

屏幕渐黑，心电图拉平，持续长音。浮现一行文字：

> "如果一切只是发生在游戏里，那该多好。"

然后弹出：

> "在中国，每年有数十万人死于糖尿病及其并发症。这不是游戏。"

最后显示"重新开始？"——给玩家重试的机会，也暗示现实中的人没有重来的机会。

## 五、核心玩法

### 5.1 玩法循环（Core Loop）

```
进入关卡 → 躲避弹幕 / 障碍 → 击毁敌人获得"血糖能量" →
收集道具强化 → 积累大招能量 → 击败关卡 Boss →
弹出科普档案卡 → 结算 → 解锁图鉴 → 进入下一关
```

### 5.2 即时玩法体验

1. **密集弹幕规避**：玩家在敌我双方的弹幕中穿梭，依靠精准走位躲避攻击。命中判定盒比视觉模型小（STG 式宽容判定），简单档进一步缩小判定盒。
2. **击杀与收集**：自动开火持续输出，击毁敌人获得分值、道具掉落和大招充能。玩家需要在高密度弹幕中判断"该走位还是该抢道具"的瞬时决策。
3. **资源管理**：大招"胰岛素脉冲"是一个"关键拯救"按钮——需要积累足够的能量才能释放，释放时机（保命 vs 等 Boss 暴露弱点时集火）考验策略判断。
4. **阶段性挑战**：每关分为导入段（教学/低强度）→ 推进段（杂兵波次+精英）→ Boss 段（机制性决战），节奏螺旋上升。

### 5.3 情绪曲线设计

```
紧张度
 ↑                                       ┌── Boss3（决死一战）
 │                          ┌── Boss2 ───┘
 │            ┌── Boss1 ────┘
 │   ┌─────────┘   +精英
 │ ──┘  教学
 └─────────────────────────────────────────→ 关卡进度
   关1导入  关1推进  关2       关3
```

玩家在每关的"导入段"获得喘息与剧情，在"推进段"逐步升温的压力中积累战力，在"Boss 段"释放全部积累（火力+大招）进行决战。通关后弹出科普卡——节奏从紧张陡然转为学习与反思，形成"战→学→战"的独特节奏。

### 5.4 操作方案

| 操作 | 键盘 | 手柄 | 触屏（预留） |
| --- | --- | --- | --- |
| 移动 | WASD / 方向键 | 左摇杆 | 拖拽战机 |
| 开火 | 自动开火（按住空格强化） | 自动 / RT | 自动 |
| 大招"胰岛素脉冲" | Shift / J | LB | 屏幕右侧按钮 |
| 暂停 | Esc | Start | 暂停按钮 |

> **设计原则**：核心仅"移动 + 大招"两件事，符合雷霆战机式的低门槛。开火默认自动。

### 5.5 道具系统（科普隐喻）

| 道具 | 效果 | 科普隐喻 |
| --- | --- | --- |
| 胰岛素结晶（蓝） | 火力 +1 级 | 胰岛素帮助控糖 |
| 膳食纤维核心（绿） | 恢复护盾 | 高纤维饮食有益血糖 |
| 运动能量（橙） | 移动速度 +、暂时无敌 | 运动提升胰岛素敏感性 |
| 高糖陷阱（红，负面） | 误吃后短时火力下降 / 屏幕泛红 | 高糖饮食加重病情（警示） |

> "高糖陷阱"故意做成诱人的甜甜圈 / 奶茶造型，玩家本能想吃，吃了却受罚——用玩法本身完成"高糖有害"的警示教育。

## 六、特色设置

### 6.1 特色概述

1. **题材即玩法**：游戏中的每一个敌人、Boss、道具、地图、机制，全部由真实的糖尿病病理知识转译而来，玩家在战斗中自然习得医学常识。
2. **三层科普闭环**：通关弹出"病理档案"卡片（核心）→ 死亡读条显示冷知识（辅助）→ 主菜单"医学档案库"收集图鉴（深度），形成完整学习链路。
3. **科幻恐怖的人体内景**：3 个关卡对应 3 种截然不同的视觉主题——暗红黏腻的血管隧道、墨黑出血的眼底世界、黄绿坏死的胰岛工厂——全部以分层视差滚动呈现。
4. **三档难度适配全人群**：简单模式（科普优先，面向课堂/长辈）→ 普通模式（标准射击体验）→ 困难模式（弹幕地狱，面向硬核玩家）。
5. **数据驱动架构**：所有敌人、关卡、弹幕、难度参数均通过 ScriptableObject 配置 + CSV 导入，策划可在 Unity Inspector 中调节一切而不改一行代码。
6. **双结局警示设计**：真结局（通关→患者苏醒→"真正的防线是你自己的生活方式"）+ 坏结局（失败→心电图拉平→"如果一切只是发生在游戏里"）。

### 6.2 编辑器

本作内置数种编辑器工具，以支持快速迭代与内容填充：

- **波次配置导入工具**：策划在 CSV 中填表（关卡波次配置表、敌人属性表、Boss 配置表、难度系数表），通过 Unity Editor 菜单一键导入为 ScriptableObject 资产。支持校验（敌人 ID 存在性、时间递增性、数值合法性）并报错定位到行号。
- **弹幕编辑器**：在 Inspector 中可视化调节弹幕参数（子弹数量、角度、速度、扩散范围、连射次数），即时预览弹幕形态。
- **关卡时间轴编排**：通过 LevelTimeline 资产以时间轴形式编排每关的敌人波次、Boss 出场、补给段、环境事件，全程可视化。

## 七、游戏世界与关卡设计

> **设计要求落实**：不同地图的关卡 + 简单 / 普通 / 困难三档难度。

### 7.1 物理世界概述

游戏世界设定在患者"林先生"的体内微观环境。整个世界的物理规则在以下维度做了游戏化处理：

- **尺度**：正常人体细胞大小为 10-100μm，但在游戏中，战机（纳米尺度）被表现为屏幕约 3-4% 的面积（约 128×128 像素区域），而血管内径被放大为全屏宽度（约 1080px）。这是一种"超微缩"视角——玩家看到的是一个被放大到可辨识尺度的微观世界。
- **重力**：液体环境中重力感弱化。战机在血浆/组织液中悬浮移动，因此移动惯性极小（瞬停瞬走），符合 STG 的操作手感需求。
- **流体感**：环境中的"心跳搏动"（背景节律性缩放）、血细胞的漂流、血雾的扩散——所有这些细节共同营造在液体中运动的沉浸感，但并不影响核心的射击判定。

### 7.2 关卡总览（3 张不同地图）

| 关卡 | 地图主题 | 视觉关键词 | 对应病理 | 时长 |
| --- | --- | --- | --- | --- |
| **第一关：血潮** | 血管 / 血液 | 暗红血流、漂浮血球、糖化结晶痂 | 高血糖、血液黏稠 | 约 3-4 分钟 |
| **第二关：盲视** | 眼底 / 视网膜 | 渗血的微血管、神经网、闪烁视野 | 糖尿病视网膜病变 | 约 3-4 分钟 |
| **第三关：核心** | 胰腺 / 胰岛 | 衰竭的胰岛工厂、坏死组织 | 胰岛功能衰竭 | 约 4-5 分钟 |

### 7.3 关卡通用结构

每关分为三段式节奏：

```
[导入段] 剧情 + 教学/低强度小怪
   ↓
[推进段] 杂兵波次 + 障碍 + 道具 + 1 个精英怪
   ↓
[Boss 段] 关卡 Boss 战 → 通关 → 科普档案卡
```

### 7.4 第一关《血潮》详细设计

- **场景**：纵向滚动的暗红血管隧道，血浆中漂浮变异红细胞，血管壁挂满黄白色糖化痂与尖刺状糖结晶。远处血管分叉的阴影在血雾中隐约可见。
- **环境危害**：周期性收缩的血管壁（挤压通道，需走位）；漂浮的糖晶障碍物（可击碎）。
- **敌人**：糖晶虫（直线冲锋）、变异白细胞（吐酸性子弹）。
- **精英怪**：血栓核（静止，召唤小怪）。
- **Boss**：**黏稠之主（高血糖凝块）**——见 [第八章 敌人与 Boss 设计](#八敌人与-boss-设计)。
- **科普锚点**：通关卡片讲"什么是高血糖 / 正常空腹血糖范围 3.9-6.1 mmol/L"（K02）。
- **情绪**：幽闭、黏腻、搏动的压迫感。玩家能"听到"放大的心跳声在血管中回荡。

### 7.5 第二关《盲视》详细设计

- **场景**：眼球内部，背景是放射状视网膜神经纤维网如墨黑蛛网，破裂的微血管不断喷出血雾，遮挡视野。白色的"软性渗出"（棉绒斑，视网膜缺血的标志）如幽灵般漂浮。
- **环境危害**：**视野遮挡**——血雾会短暂遮住屏幕局部（既是恐怖氛围又是真实难度——这正是糖尿病视网膜病变患者真实体验的游戏化转译）；漂浮的渗出物；新生异常血管丛缠绕，触碰即减速。
- **敌人**：出血点（爆裂溅射弹幕）、新生异常血管（缠绕型，限制走位）。
- **Boss**：**血网之眼（视网膜病变）**——见 [第八章 敌人与 Boss 设计](#八敌人与-boss-设计)。
- **科普锚点**：通关卡片讲"糖尿病视网膜病变是成人失明主因之一"（K04）。
- **情绪**：黑暗、惊悚、失明恐惧。玩家突然看不清屏幕的某个区域。

### 7.6 第三关《核心》详细设计

- **场景**：胰腺深处的胰岛"工厂"——巨大的细胞结构体排列成类似工业设施的外观，半数已坏死停转（灰色/黑色坏死组织），残存的胰岛 β 细胞发出微弱的青蓝色荧光（象征仅存的胰岛素分泌能力）。地面有腐蚀性的高糖渗出液形成发绿毒云，组织碎块从天花板上不断崩落。
- **环境危害**：坏死区的**腐蚀云**（持续掉血区域）；**崩落的组织碎块**（触碰受伤）；高糖孢子增殖（不清会越来越多——隐喻"病情拖延的代价"）。
- **敌人**：吞噬细胞群、高糖孢子（增殖型，不清会越来越多）。
- **Boss**：**胰岛吞噬者（终极 Boss，三阶段）**——见 [第八章 敌人与 Boss 设计](#八敌人与-boss-设计)。
- **科普锚点**：通关卡片讲"预防为主：管住嘴、迈开腿、定期监测"（K08）。
- **情绪**：衰败、绝望、决战前的悲壮。这是身体最后的防线。

### 7.7 缩放比例

游戏采用**抽象化的超微缩比例**表现微观世界：

| 对象 | 在现实中的实际尺度 | 游戏中的屏幕占比 |
| --- | --- | --- |
| 玩家战机（纳米机器人） | ~100 nm | 约 3-4%（128px 宽区域） |
| 红细胞 | ~7 μm | 约 1-2%（背景漂浮物） |
| 血管内径 | ~5-10 μm（毛细血管） | 全屏宽（1080px） |
| 糖晶虫（敌） | 抽象表现 | 约 3-4% |
| Boss（病理凝块） | 抽象表现 | 约 20-30% |

这种缩放并非严格尊重现实比例，而是以**可读性与游戏性优先**：微观实体被放大到可辨认的尺寸，以便玩家在高速卷轴中做出判断。

### 7.9 环境状态（室内气候）

本作没有传统意义上的"天气"与"昼夜"系统——因为游戏场景在人体内部，不受外部气象影响。但每一关有其独特的**环境状态**：

- **关一 · 血潮**：血浆的暗红"雾霾"弥漫，时而被心跳搏动扰动。
- **关二 · 盲视**：血雾如"暴雨"般间歇喷涌，造成局部"风暴"遮挡。
- **关三 · 核心**：腐蚀性毒云如"毒雾"低悬，病变组织的"酸雨"碎块不断坠落。

### 7.10 时间感

游戏中的"时间流逝"体现为：

- 关卡底部的心电图进度条——象征林先生的生命体征，随关卡推进而波动。
- 玩家在关卡中的进度 = 纳米战机深入体内的旅程 = 一次与时间赛跑的抢救手术。
- 整局游戏实际通关时长（普通难度）：约 12-15 分钟（三关合计）。
- 每关有固定的时间轴长度，不可能"回头"——这隐喻着：**治疗糖尿病没有回头路，拖延一天，伤害就加深一分。**

### 7.11 世界规划层级结构

```
糖战·胰岛防线 — 游戏世界
├── 第一关《血潮》— 血管战区
│   ├── 导入段：注射针管入口 → 血管前段
│   ├── 推进段：血管中段（波次 W01–W08）

### 8.2 Boss 设计

#### Boss 1 — 黏稠之主（高血糖凝块）

- **关卡**：第一关《血潮》
- **形态**：巨大的半透明黏稠凝块，内部悬浮无数糖晶。晶核弱点发出毒糖紫光。
- **阶段数**：2（完整态 → 分裂态）
- **机制**：
  - 完整态：大范围黏液弹幕覆盖 + 周期吐出减速黏液。
  - **分裂**：被打到半血时分裂成多个小凝块（隐喻血糖失控扩散），每个小凝块独立攻击。
  - 需在小凝块重新聚合前逐一击破。
- **理想战斗时长**：45-60 秒（普通难度）
- **科普点（K02）**：高血糖使血液黏稠、循环受阻。正常空腹血糖 3.9-6.1 mmol/L。
- **关联数据表**：[Boss配置表.csv](配置表/Boss配置表.csv) — Boss1 行

#### Boss 2 — 血网之眼（视网膜病变）

- **关卡**：第二关《盲视》
- **形态**：布满破裂血管的巨大眼球，瞳孔即弱点。新生血管如触须环绕。
- **阶段数**：2（护盾态 → 暴露态）
- **机制**：
  - 护盾态：新生血管再生护盾环绕，需先打断护盾（击中新生血管）才能攻击瞳孔弱点。
  - **周期"流血致盲"**：屏幕被红色血雾大面积遮挡 2-3 秒（既是恐怖氛围也是真实难度）。
  - 暴露态：护盾被打断后瞳孔弱点暴露，此时为输出窗口。
  - 放射状血雾弹幕持续全场。
- **理想战斗时长**：50-70 秒（普通难度）
- **科普点（K04）**：糖尿病视网膜病变可致盲，需定期眼底检查。
- **关联数据表**：[Boss配置表.csv](配置表/Boss配置表.csv) — Boss2 行

#### Boss 3 — 胰岛吞噬者（终极 Boss）

- **关卡**：第三关《核心》
- **形态**：盘踞在胰岛工厂上的巨型高糖怪核，象征糖尿病本体。崩解期暴露明亮的内核。
- **阶段数**：3
- **机制**：
  1. **侵蚀期**（100%-70% HP）：弹幕压制 + 召唤高糖孢子。孢子不清会增殖。
  2. **狂暴期**（70%-30% HP）：吞噬残余胰岛细胞回血（每次恢复 5% HP），玩家需优先攻击吞噬触手，保护发光胰岛点不被吸入。
  3. **崩解期**（30%-0% HP）：核心完全暴露，但弹幕密度达到峰值。需在限定时间（45 秒）内集火击杀——配合"胰岛素脉冲"大招打爆发。
- **理想战斗时长**：60-90 秒（普通难度）
- **科普点（K08）**：胰岛功能一旦严重衰竭难以逆转，预防远胜治疗。均衡饮食、规律运动、控制体重、定期监测。
- **关联数据表**：[Boss配置表.csv](配置表/Boss配置表.csv) — Boss3 多行（每阶段一行）

### 9.2 设计理念

- **简单 = 科普优先**：面向健康教育课堂、社区讲座、家长/长辈试玩。重点是"看懂知识 + 轻松通关"，避免因难度劝退非玩家群体。
- **普通 = 平衡体验**：标准的雷霆战机式爽快弹幕射击。敌我平衡，有挑战但不至于卡关——玩家全程不升级也能在合理操作下通关。
- **困难 = 硬核弹幕地狱**：面向 STG 老玩家的终极挑战。弹幕密度接近"东方"系列水平，但需保证弹幕有"安全缝隙"（手测验证可通）。

### 9.3 难度曲线（普通档）

```
强度

 │            ┌── Boss1 ────┘
 │   ┌────────┘

   关1导入  关1推进  关2      关3

### 9.4 动态难度调整（DDA，可选）

普通模式下启用轻量 DDA：玩家在同一段落连续死亡 ≥2 次时，该段弹幕密度临时降低 10%，避免卡关劝退。开关可在设置中关闭。

## 十、科普内容设计

> **设计要求落实**：必须有系统化的科普内容设计。本作的科普采取"**沉浸隐喻 + 显性卡片**"双层结构，确保玩家既能在玩法中体会，也能明确记住知识点。

### 10.1 科普三大触发场景

1. **关卡通关卡片**（核心层）：每关结束弹出"病理档案"知识卡片，图文讲解 1-2 个核心知识点。简单难度强制阅读（停止滚动），普通/困难可跳过。
2. **死亡 / 受伤提示**（辅助层）：死亡读条界面随机显示一条糖尿病小知识（变等待为学习）。
3. **道具 / 敌人图鉴**（深度层）：主菜单"医学档案库"收集解锁，每个敌人对应真实病理解释。全收集约需通关普通难度 1-2 次。

## 十二、用户界面设计

### 12.1 概述

整体 UI 风格定位为 **"ICU 医疗监护仪"美学**——半透明深色玻璃面板 + 青色霓虹描边（`#2CA8FF`）+ 科技/等宽字体 + 生命体征监控仪的视觉语言。让玩家感觉自己不是在玩射击游戏，而是在**执行一台精密抢救手术**。

### 12.2 HUD（游戏内抬头显示）

- **左上区域**：生命值条（绿色渐变为红色）+ 护盾值条（蓝色），样式模拟心电监护仪的生命体征波形。
- **左下区域**：大招"胰岛素脉冲"能量槽——满时为明亮青白色，未满为暗色，外圈有发光刻度环。
- **右上区域**：当前得分（7 段数码管字体，闪烁效果）。
- **底部**：心电图式关卡进度条——代表林先生的生命体征，随关卡推进向右推移。在 Boss 战时明显波动（加速闪烁），营造紧张感；在通关恢复时趋于平稳。
- **风格细节**：所有 HUD 元素均为半透明深色底（`rgba(0,0,0,0.4)`）+ 青色霓虹描边，荧光感。不挡视野。

### 12.3 主菜单

- 黑色背景上浮现隐隐搏动的血管纹理。
- 菜单选项垂直排列：**开始游戏 / 选择关卡 / 选择难度 / 医学档案库 / 设置 / 关于**。
- 选项选中时 ICU 监护仪式的高亮描边 + 右侧出现心电图短波形。
- 右下方显示版本号与免责声明入口。

### 12.4 难度选择界面

- 三个选项做成**血糖风险等级**视觉隐喻：
  - 简单（绿色，`Safe`，血糖正常范围图标）——"科普模式：轻松通关，重点学习"
  - 普通（黄色，`Warning`，前驱糖尿病图标）——"标准体验：平衡挑战与科普"
  - 困难（红色闪烁，`Danger`，高血糖危险图标）——"生存模式：弹幕地狱，硬核挑战"

### 12.5 结算界面

## 十三、武器系统与数值设计

### 13.1 玩家战机（INS-09）基础属性

- **生命值（HP）**：简单 5 / 普通 3 / 困难 1
- **护盾值**：通过拾取"膳食纤维核心"恢复
- **移动速度**：基础 8 单位/秒（屏幕基准 1080px），拾取"运动能量"后 ×1.5 持续 5 秒
- **火力等级**：Lv1-Lv5（通过拾取"胰岛素结晶"升级）

### 13.2 主武器：胰岛素结晶机枪（自动开火）

游戏全程自动开火，玩家无需按射击键。火力等级通过拾取"胰岛素结晶"道具提升，受伤死亡后火力回退 1 级（而非归零——惩罚但不过度挫败，隐喻"血糖反弹"）。

| 火力等级 | 弹道描述 | 升级所需结晶数 | DPS 系数 | 开火间隔 |
| --- | --- | --- | --- | --- |
| **Lv1** | 单发直射（基础弹） | 初始 | ×1.0 | 0.25s |
| **Lv2** | 双发平行弹 | 1 个 | ×1.8 | 0.22s |
| **Lv3** | 三向散射（中 1 + 左右 30° 各 1） | 2 个 | ×2.5 | 0.20s |
| **Lv4** | 五向散射（中 1 + 左右 15°/30° 各 1） | 3 个 | ×3.5 | 0.18s |
| **Lv5** | 五向散射 + 2 枚自动追踪弹 | 4 个 | ×5.0 | 0.15s |

> 子弹外观：青白结晶能量弹（`#2CA8FF`），发光，与敌人毒糖紫（`#B026FF`）/血红（`#D00000`）的弹幕形成清晰的颜色区分。

### 13.3 大招武器：胰岛素脉冲（Insulin Pulse）

**胰岛素脉冲**是本作的终极技能（"必杀/炸弹"），象征"注射胰岛素快速降糖"的医疗行为。

- **充能方式**：击杀敌人积累能量（每击杀 +1 格，满格 100），Boss 击杀 +20 格，拾取胰岛素结晶额外 +10 格。
- **效果**：释放一圈青白结晶能量冲击波向外扩散——
  - 清除全屏敌方子弹（"清屏"效果）。
  - 对范围内所有敌人造成相当于 30 次普攻的伤害。
  - 对 Boss 造成额外弱点伤害（破盾 / 打断阶段技能）。
- **冷却/限制**：
  - 能量条满后（100 格）才能释放，释放后清空重新积累。
  - 简单难度充能速度 ×1.5（约 15 秒满），普通标准 ×1.0（约 25 秒），困难 ×0.6（约 40 秒）。
- **视觉表现**：1024×1024 环形冲击波序列帧（8-10 帧），青白色（`#2CA8FF` → `#FFFFFF`）光芒扩散，带结晶碎片粒子。

### 13.4 计分系统

```
关卡得分 = 击杀分 + 收集分 + 通关时间奖励 + 难度系数加成 - 受击扣分

击杀分 = Σ(每个敌人的分值)          ← 见第八章杂兵分值
收集分 = 拾取道具数 × 50
通关时间奖励 = max(0, (关卡基准秒数 - 实际通关秒数) × 10)
难度系数加成 = (击杀分 + 收集分 + 时间奖励) × (难度系数 - 1.0)
受击扣分 = 受击次数 × 30

难度系数：简单 ×1.0 / 普通 ×1.5 / 困难 ×2.5
```

排行榜（本地 PlayerPrefs）按总分降序排列，鼓励重玩与高难挑战。

### 13.5 数值平衡原则

1. **普通档可通性**：玩家全程不升级（Lv1）也能在合理操作下通关，升级则更轻松（Lv5 通关时间约缩短 40%）。
2. **Boss 血量反推公式**：`Boss_HP = 理想DPS(Lv3) × 期望战斗时长 × 难度血量系数`
   - Boss1 理想时长 45-60s → HP ≈ 基准 ×1.0
   - Boss2 理想时长 50-70s → HP ≈ 基准 ×1.2
   - Boss3 理想时长 60-90s → HP ≈ 基准 ×1.5
3. **弹幕安全缝隙**：所有弹幕保证存在 ≥ 玩家判定盒 3 倍宽的"安全缝隙"，困难档 ≥ 2 倍。不做无解弹幕。
4. **道具掉落平衡**：普通难度下平均每 8-12 个敌人掉落 1 个道具，保证全程约 15-20 次升级机会。

## 十五、渲染系统

### 15.1 概述

本作采用 **Unity URP（Universal Render Pipeline）2D Renderer** 进行渲染。整体画面追求生物科幻恐怖电影质感的数字绘画效果。

### 15.2 2D 渲染方案

- **渲染方式**：2D Sprite 为主（所有角色、敌人、子弹、道具、背景均为 Sprite），辅以 Unity 粒子系统（特效）。
- **视差滚动**：每关背景分为 3 层（远景 / 中景 / 前景），以不同速度纵向滚动，营造纵深感。层间速度比为 0.3 : 0.6 : 1.0。单层建议 ≥ 1080×1920 或可平铺条带。
- **全局后处理（URP Volume）**：
  - **暗角（Vignette）**：画面四角渐暗，强度 0.35-0.45，强化幽闭感与聚焦中心。
  - **泛光（Bloom）**：青色霓虹、毒糖紫等发光元素渗晕，阈值 0.8，强度 0.3-0.5，增强诡异氛围。
  - **胶片噪点（Film Grain）**：轻量噪点叠加（强度 0.15），模拟纪录片/恐怖电影质感。
  - **色调映射（Color Grading）**：统一压暗高光、加深阴影，低饱和中有高饱和"危险色"对比。Lift 偏冷（RGB +0.02），Gamma 偏暗（×0.95），Gain 保持不变。

### 15.3 视角

- **主视角**：俯视（Top-Down），摄像机固定正上方（Z=-10），跟随玩家 Y 轴平滑移动（X 轴锁定，营造纵版卷轴感）。
- **摄像机行为**：背景以恒定速度向下滚动（驱动关卡推进感），玩家可自由在屏幕范围移动（X/Y 均受边界限制，边界 = 屏幕 ±5% 缓冲）。
- **特殊视角偏移**：
  - Boss 战开场时摄像机短暂拉远（FOV 等效由 60→50，持续 1.5 秒），展示 Boss 全貌。
  - 释放"胰岛素脉冲"大招时摄像机轻微震动（Shake 振幅 0.3，频率 20Hz，持续 0.5s）+ 短暂闪白。
  - 第二关"血雾致盲"时，摄像机局部增加红色半透明遮挡层（程序动态控制透明度与位置）。

## 十六、游戏引擎架构

### 16.1 概述

游戏基于 **Unity 2022 LTS** 引擎开发，利用其成熟的 2D 工具链与 URP 后处理管线。核心架构遵循"**数据驱动 + 事件解耦 + 对象池优先**"三大原则，以 ScriptableObject 作为配置中枢，实现策划调参零代码。

### 16.2 核心系统模块

```
GameManager        —— 全局状态机（Boot/Menu/Playing/Paused/Result）、难度参数注入
LevelManager       —— 关卡时间轴、段落推进、背景滚动控制
SpawnSystem        —— 基于时间轴的波次生成（数据驱动）
BulletPool         —— 子弹对象池（性能关键，杜绝运行时 Instantiate）
EnemyAI            —— 敌人行为（移动模式库 + 弹幕发射器组合）
BulletPattern      —— 弹幕发射器（角度/数量/速度参数化，Inspector 可调）
PlayerController   —— 移动 / 自动开火 / 大招 / 受伤 / 死亡
PowerUpSystem      —— 4 种道具拾取与效果（含负面陷阱）
BossController     —— Boss 多阶段状态机（3 个 Boss 统一框架复用）
KnowledgeSystem    —— 科普卡片/图鉴/死亡提示/免责声明（数据驱动）
UIManager          —— HUD / 菜单 / 结算 / 卡片 / 档案库
AudioManager       —— BGM / SFX / 低血量警报（AudioMixer 分组）
SaveSystem         —— 进度 / 解锁 / 排行榜（PlayerPrefs / JSON）
```

### 16.3 数据驱动架构

用 `ScriptableObject` 配置一切可调参数：

- `EnemyData`：血量、速度、弹幕 pattern、掉落、对应科普 ID。
- `WaveData / LevelTimeline`：何时何处刷什么怪，与 [关卡波次配置表.csv](配置表/关卡波次配置表.csv) 一一对应。
- `BulletPatternData`：弹幕参数（角度/数量/速度/扩散/连射），让策划可在 Inspector 调弹幕，不改代码。
- `DifficultyData`：简单/普通/困难全参数（12 项），与 [难度系数表.csv](配置表/难度系数表.csv) 对应。
- `ItemData`：道具类型/效果数值/持续时间/图标/拾取音效。
- `KnowledgeCard`：编号/标题/正文/插画/触发类型/关联关卡或敌人 ID。

> **好处**：调难度/调关卡/加科普都改数据，不改代码，2 周内可快速迭代。

### 16.4 CSV → ScriptableObject 数据流

```
策划填写 CSV（4 张表）
  → Unity Editor 导入工具（菜单 Tools/导入XX配置 一键执行）
  → 自动解析为 ScriptableObject 资产
  → 校验（敌人ID存在性、时间递增性、数值合法性）→ 报错定位到行号
  → 运行时各系统读取 SO 配置
  → 策划改 CSV → 重新导入 → 不改代码
```

### 16.5 碰撞检测

碰撞检测采用 **2D Trigger + 简化判定盒**方案：

- 玩家的命中判定为一个半径为战机本体 40% 的**小圆形 Trigger**（比视觉模型小得多，符合 STG 宽容判定的传统）。
- 简单难度下判定盒进一步缩小至 30%（系数 ×0.7），容错更高。
- 敌人的命中判定为一个覆盖本体核心的矩形/圆形 Trigger。
- 敌我子弹分属不同的 Physics2D Layer：`Player`、`PlayerBullet`、`Enemy`、`EnemyBullet`、`Boss`、`Item`、`Environment`。在碰撞矩阵中仅勾选必要的跨层碰撞：`PlayerBullet×Enemy`、`PlayerBullet×Boss`、`EnemyBullet×Player`、`Player×Item`，极大减少无效检测。
- 所有子弹碰撞使用 `OnTriggerEnter2D` 而非 `OnCollisionEnter2D`（后者涉及更昂贵的物理模拟）。

### 16.6 性能要点

- 子弹、敌人、特效全部走**对象池**（预实例化 + 复用），运行时零 `Instantiate` / `Destroy`。
- 同屏子弹数设硬上限（500 发），超出时不再生成新子弹，防止极端弹幕场景卡顿。
- 背景使用分层条带纹理平铺（而非整个大贴图），减小显存占用。
- 所有 Sprite 打入 **Sprite Atlas**，降低 Draw Call。
- 目标帧率：60 FPS 稳定（普通难度、中配 PC）。
- `FixedUpdate` 仅用于物理判定，游戏逻辑全部走 `Update`（`Time.deltaTime` 保证帧率无关）。

### 16.7 流体表现（视觉层）

游戏世界中没有传统意义上的"水"，但流体感贯穿始终——血浆的黏稠流动（背景视差 + 粒子系统）、血雾的扩散（动态半透明叠加层）、腐蚀云的翻滚（Shader 动画）。这些流体表现并非物理模拟，而是**视觉层**的动画 + 特效，服务于恐怖氛围而非真实流体力学。

## 十七、照明模式

### 17.1 概述

由于游戏场景在人体内部（无自然光），所有"光源"均为自发光物体——战机引擎的冷蓝辉光（`#2CA8FF`）、敌人/Boss 的高糖紫色（`#B026FF`）/荧光绿色（`#39FF14`）生物发光、胰岛细胞的微弱青色生命之光。

### 17.2 2D 自发光的假照明

并非 3D 光照系统，而是在 Sprite 上叠加发光材质（HDR 颜色 + URP Bloom 后处理），营造发光体在黑暗环境中"照亮周围"的视觉错觉。

### 17.3 关键照明效果

- **受伤时的边缘红光**：当玩家血量低于 30% 时，屏幕边缘叠加红色半透明遮罩（`#D00000`，alpha 0.2→0.4）+ 轻微脉动闪烁（周期 1s），模拟"失血/危重"的紧迫感。
- **Boss 暴露弱点时的亮度变化**：Boss 弱点部位在可攻击时材质切换为高亮版本（HDR Emission 值 ×2.0），清晰指示攻击窗口；暴露倒计时 ≤5 秒时闪烁加速。
- **难度档位的视觉提示**：简单难度 HUD 边框为绿色（`Safe`），普通为黄色（`Warning`），困难为红色闪烁（`Danger`）——将糖尿病风险等级可视化融入 UI 照明。

## 十八、角色文字人设

### 18.1 INS-09（玩家战机）—— "最后的希望"

- **物理外观**：长约 100nm 的流线型纳米飞行器，俯视视角下呈细长三角箭头造型（灵感来自注射器针尖 + 结晶体切割面）。机身主体为抛光镀铬银白与医疗白，表面蚀刻冷蓝色发光能量线（`#2CA8FF`）。中心嵌一颗拇指大小的胰岛素结晶体——发出柔和的青白色脉动光（象征"生命之源"）。引擎尾焰为两道青蓝渐变粒子流。
- **性格/特征**：无机生命。它的"性格"来自搭载的 AI 瑞吉娜——理性、冷静、在危急时刻也会流露出一丝"决绝"的语调变化。INS-09 没有台词，它的存在本身就是"希望"的象征。
- **设计隐喻**：INS-09 = 外源胰岛素。在糖尿病患者体内，外源胰岛素是"外来的拯救者"——正如 INS-09 这艘来自体外的人工纳米战机。它的冷蓝色金属质感，与周围有机腐败的暗红/毒糖紫色环境形成强烈反差——干净、锋利、精准，是这个腐坏世界里唯一有序的存在。

### 18.2 瑞吉娜 REGINA —— "冷静的导航者"

- **外观**：全息投影的 AI 头像——由青色数据粒子（`#2CA8FF`）构成的女性面孔，半透明（alpha ~0.7），部分面部区域化为流动的代码/波形。不完全拟人——保留"机器"的冷感。周围环绕悬浮的环形 HUD 元素（心率波形、化学分子式、导航矢量）。
- **性格**：临床般的冷静与精确，但在关键剧情节点（如抵达已经大片坏死的胰岛时），语调会出现极微弱的波动——"INS-09……这比预计的更糟。"

### 18.3 林先生 —— 沉默的主角（患者）

- **人物标签**：47 岁男性，办公室职员。体重超标（BMI 约 29）。常年久坐、嗜甜、三餐外卖、从不体检。他从未在游戏中"出场"（玩家始终在体内），但他的**生命体征**贯穿全程——心电图进度条、低血量时的警报、通关时的苏醒。
- **叙事功能**：林先生是"每一个可能成为糖尿病患者的人"的符号化身。他的具体性（有年龄、有职业、有生活习惯）让人产生"这不就是我/我爸爸/我丈夫吗"的代入感；他的沉默（全程昏迷）让人始终关注他的性命本身。

### 19.3 场景组织（省事方案）

只做 **1 个 `Gameplay` 场景**，通过加载不同 `LevelTimeline` SO 数据切换关卡内容，避免为每个关卡重复搭场景。推荐 2 周项目采用此方案。

### 19.4 关键数据流

**关卡启动流程**：
```
GameManager(选难度→加载Gameplay场景)
  → LevelManager.Init(读取 LevelTimeline + 注入 DifficultyData)
  → BackgroundScroller 启动（3层视差）
  → SpawnSystem 按时间轴推进
      └ 到达波次触发时间 → 从 ObjectPool 取敌人 → 按 EnemyData 初始化（难度系数已注入）
  → 到达 Boss 段 → LevelManager 激活 BossController
  → Boss 死亡事件 → LevelManager 触发"通关"
  → KnowledgeManager 弹病理档案卡 → ScoreManager 结算 → UIManager 显示结算
```

**一次击杀的事件链**：
```
PlayerBullet 命中 Enemy
  → Enemy 扣血；死亡时：EventBus.Publish(OnEnemyKilled)
      ├ ScoreManager 加分
      ├ DropTable 概率生成道具（从池）
      ├ UltimateSkill 充能（+1 格，Boss 击杀 +20 格）
      ├ VFX 播放爆炸（从池）
      └ KnowledgeManager 记录图鉴解锁
  → Enemy 归还对象池
```

**难度注入路径**：
```
主菜单选难度 → GameManager.CurrentDifficulty = DifficultyData
  → 各模块启动时向 DifficultyService 查询系数：
      PlayerHealth(初始生命) / BulletSpawner(密度·弹速) / EnemyBase(血量) /
      DropTable(掉落率) / UltimateSkill(充能速度) / PlayerHitbox(判定盒) /
      LevelManager(存档点) / KnowledgeManager(卡片是否强制)
```

### 19.5 命名与编码规范

- **类名**：PascalCase（`PlayerController`）；**字段/局部变量**：camelCase；**常量**：UPPER_CASE。
- **私有序列化字段**：`[SerializeField] private` + camelCase，便于 Inspector 调参又不暴露 public。
- **一个文件一个类**，文件名 = 类名。
- **避免 Update 滥用**：能用事件/协程的不轮询；大量对象优先考虑统一管理器集中 tick。
- **禁止运行时 Instantiate 子弹/敌人/特效**：一律走对象池。
- **MonoBehaviour 只做表现与生命周期**，纯逻辑/数据尽量放普通类或 SO，便于测试。
- **注释**：只解释"为什么"与非显而易见的约束，不写复述代码的废话注释。
- **美术资产命名**：`类别_名称_状态_序号`，例：`enemy_sugarworm_idle_00`、`bg_l1_blood_far`、`ui_btn_normal`。

## 结语

> 克里斯·泰勒（Chris Taylor）在游戏策划模板的末尾写道：
>
> *"我希望可以用更多的时间来书写这个文档模板，并希望将它做成一个集体设计游戏的路标。不幸的，它花去了我太多的时间，而时间又是工作中所缺少的。我想你一定可以有所启发。而且你不需要记住那些信息的排列顺序，我只是恰巧地按照这个顺序想到这些而已。"*
>
> 在这个策划文档中，我尝试用 Chris Taylor 留下的模板框架，融合系统化的数值设计与技术方案，去讲述一个不那么传统的游戏策划故事——一个关于"游戏能否做更多事情"的探索。
>
> 如果《糖战·胰岛防线》有朝一日能够从这份文档中走出来，成为一个真正可玩的游戏，我希望能有玩家在通关之后，默默放下了手中的奶茶，预约了一次体检。
>
> 那就是这个游戏存在的全部意义。

**祝你好运。**
