## 一、美术资源问题 — "太丑了"

### 问题描述

当前所有图形完全由 `ProceduralSprites.cs` 代码生成（纯色几何形：圆/三角/六边形/菱形/方框），没有贴图、没有粒子特效、背景是随机彩色圆点。虽然 README 说这是"占位图保证开箱即跑"，但视觉确实过于简陋。

### 根因代码

| 文件 | 行号 | 说明 |
|------|------|------|
| [ProceduralSprites.cs](Assets/Scripts/Core/ProceduralSprites.cs) | 17–48 | 用 `Texture2D.SetPixel` 逐个像素画纯色几何形 |
| [BackgroundScroller.cs](Assets/Scripts/Level/BackgroundScroller.cs) | 24–25 | 背景是随机彩色圆形 |
| [UIFactory.cs](Assets/Scripts/UI/UIFactory.cs) | 48 | UI 按钮/面板用白色 8×8 方块的 `SolidBar()` |
| [PlayerFactory.cs](Assets/Scripts/Player/PlayerFactory.cs) | 15 | 玩家是蓝色三角形 + 橙色三角形尾焰 |
| [PrefabFactory.cs](Assets/Scripts/Core/PrefabFactory.cs) | 46–81 | 子弹/敌人/道具/VFX 模板全是圆形几何体 |

### 解决方案

**方案 A：替换真实美术资产（推荐，工作量中等）**
1. 收集/制作约 30 张 PNG 素材：玩家（1）、敌人（6 杂兵 + 3 精英 = 9）、Boss（3）、子弹（2 种）、道具（4）、UI 面板/按钮（4）
2. 修改 [PrefabFactory.cs](Assets/Scripts/Core/PrefabFactory.cs) 的 `BuildBullet/BuildEnemy/BuildItem`，用 `Resources.Load<Sprite>()` 加载 PNG 替代 `ProceduralSprites.Get()`
3. 修改 [PlayerFactory.cs](Assets/Scripts/Player/PlayerFactory.cs)，用真实 PNG 替代三角形
4. 修改 [BackgroundScroller.cs](Assets/Scripts/Level/BackgroundScroller.cs)，用多层视差 PNG 背景替代随机圆点
5. 修改 [UIFactory.cs](Assets/Scripts/UI/UIFactory.cs)，添加 9-slice 面板图

**方案 B：增强程序化生成（最快，不依赖外部资源）**
1. 给几何形添加描边/渐变/纹理，增大分辨率（64→128）
2. 添加简单的粒子拖尾（扩展现有 `Vfx.cs`）
3. 背景改为代码生成的网格线/星场，而不是彩色圆点
4. 不同形状用更丰富的颜色和发光效果

## 二、玩家伤害太低

### 问题描述

打一个小怪（20 血）需要 5-20 秒，精英（300 血）需要 75+ 秒。整数除法导致 Lv3 和 Lv5 伤害递增不平滑。

### 根因代码

[PlayerWeapon.cs](Assets/Scripts/Player/PlayerWeapon.cs) 第 43 行：

```csharp
int dmg = 1 + lv / 2;  // 整数除法！
```

实际伤害：
| 火力等级 | 计算 | 实际伤害 |
|----------|------|----------|
| Lv1 | 1 + 0 = 1 | **1** |
| Lv2 | 1 + 1 = 2 | **2** |
| Lv3 | 1 + 1 = 2 | **2** (∵ 3/2=1) |
| Lv4 | 1 + 2 = 3 | **3** |
| Lv5 | 1 + 2 = 3 | **3** (∵ 5/2=2) |

**对普通难度 E01 (20HP) 的击杀时间（Lv1，4 发/秒）：**
- 20 HP ÷ (4×1) DPS = **5 秒/只**

**对精英 ELITE1 (300HP) 的击杀时间（Lv1）：**
- 300 ÷ 4 = **75 秒**

### 解决方案

修改 [PlayerWeapon.cs](Assets/Scripts/Player/PlayerWeapon.cs) 第 43 行：

```csharp
// 修改前
int dmg = 1 + lv / 2;

// 修改后（方案 A：平滑递增）
int dmg = 1 + lv;  // Lv1→2, Lv2→3, Lv3→4, Lv4→5, Lv5→6
// DPS 变为原来的 2-3 倍

// 修改后（方案 B：快速起步）
int dmg = 2 + lv;  // Lv1→3, Lv5→7
// DPS 变为原来的 3-5 倍，初期体感大幅改善

// 修改后（方案 C：推荐，折中）
int dmg = 1 + lv * 2;  // Lv1→3, Lv2→5, Lv3→7, Lv4→9, Lv5→11
// 最平滑，升级有明确的正反馈
```

## 四、弹幕太难躲避

### 问题描述

多个敌人同时开火（扇形 3 发 + 环形 8/12/16 发），弹幕密度大、弹速高，血量压力又大（打不死敌方 → 更多子弹堆积），形成恶性循环。

### 根因代码

| 文件 | 行号 | 当前值 | 问题 |
|------|------|--------|------|
| [BulletPatternData.cs](Assets/Scripts/Data/BulletPatternData.cs) | 13 | 默认弹速 5f | 偏快 |
| [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) | 86 | E02: Fan3, 4f speed, 1.5s interval | 同屏 3+ 只时子弹多 |
| [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) | 106 | ELITE1: Ring8 (360°), 每 2.5s | 8 向同时 + 召唤 E01 |
| [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) | 111 | ELITE2: Ring12 (360°), 每 2s | 12 向环形弹幕 |
| [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) | 116 | ELITE3: Ring16 (360°), 每 2.2s | 16 向 + 召唤孢子 |
| [Bullet.cs](Assets/Scripts/Bullets/Bullet.cs) | 44 | 敌方弹半径 0.14 | 小到看不清 |
| [PlayerHitbox.cs](Assets/Scripts/Player/PlayerHitbox.cs) | 46 | 命中盒 0.14 × 系数 | 普通/困难无容错 |

### 解决方案

**4.1 降低弹速和密度**

修改 [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) 中敌人的弹幕参数：

```csharp
// 杂兵弹速降 25%，间隔拉长 30%
Pattern(PatternType.Fan3, 3, 40f, 3.0f, ...)   // 原 speed 4f → 3f
fireInterval = 2.0f  // 原 1.5s → 2.0s

// 精英环弹：减少弹数
Pattern(PatternType.Ring8, 6, 360f, ...)   // 原 8 发 → 6 发
Pattern(PatternType.Ring12, 8, 360f, ...)  // 原 12 发 → 8 发
Pattern(PatternType.Ring16, 10, 360f, ...) // 原 16 发 → 10 发
```

**4.2 增大敌方子弹、改善可见性**

修改 [Bullet.cs](Assets/Scripts/Bullets/Bullet.cs) 第 44 行和 [BulletSpawner.cs](Assets/Scripts/Bullets/BulletSpawner.cs) 第 30 行：

```csharp
// 敌方子弹半径增大（更显眼）
float radius = team == Team.Enemy ? 0.22f : 0.14f;  // 原 0.14 → 0.22
```

**4.3 增大玩家命中盒容错**

修改 [PlayerHitbox.cs](Assets/Scripts/Player/PlayerHitbox.cs) 第 22 行或 [PlayerFactory.cs](Assets/Scripts/Player/PlayerFactory.cs) 第 46 行：

```csharp
hitbox.Setup(health, 0.10f);  // 原 0.14 → 0.10（判定更小更宽容）
// 或直接改 DifficultyData 中 hitboxFactor: Easy 0.5, Normal 0.7, Hard 0.9
```

**4.4 降低难度系数**

修改 [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) 第 151–175 行：

```csharp
// 普通难度
normal.bulletDensityFactor = 0.75f;   // 原 1.0 → 0.75
normal.bulletSpeedFactor = 0.85f;     // 原 1.0 → 0.85
normal.enemyHealthFactor = 0.9f;      // 原 1.0 → 0.9

// 困难难度
hard.bulletDensityFactor = 1.2f;      // 原 1.6 → 1.2
hard.bulletSpeedFactor = 1.1f;        // 原 1.3 → 1.1
hard.enemyHealthFactor = 1.2f;        // 原 1.4 → 1.2
```

## 五、火力升级太慢

### 问题描述

升级需要收集"胰岛素结晶"道具，但掉率低（15%-40%），且所需数量随等级递增（Lv1→2 需 1 个，Lv2→3 需 2 个，累计 10 个到 Lv5）。波次之间掉率不稳定，经常全程卡在 Lv2-3。

### 根因代码

| 文件 | 行号 | 当前值 |
|------|------|--------|
| [PlayerWeapon.cs](Assets/Scripts/Player/PlayerWeapon.cs) | 99 | `insulinCollected >= level` — 升级门槛 = 当前等级 |
| [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) | 216–248 | 关卡波次 insulin 掉率 0.15-0.5 |

### 解决方案

**5.1 降低升级门槛**

修改 [PlayerWeapon.cs](Assets/Scripts/Player/PlayerWeapon.cs) 第 99 行：

```csharp
// 修改前
if (insulinCollected >= level)

// 修改后：固定 2 个升一级
if (insulinCollected >= 2)

// 或更激进：每个结晶都升级（直到 Lv5）
if (insulinCollected >= 1)
```

**5.2 提高道具掉率**

修改 [GameDatabase.cs](Assets/Scripts/Core/GameDatabase.cs) 第 216–248 行的关卡波次掉率：

```csharp
// 原 0.3 → 0.5（50% 掉率），原 0.15 → 0.3
// 确保每波至少 1-2 个结晶
```

### 🔵 P2 — 数值调优（游戏体验）

#### P2-1: 玩家伤害公式微调

**文件：** [PlayerWeapon.cs:43](Assets/Scripts/Player/PlayerWeapon.cs)

**当前值：** `int dmg = 2 + lv;` — Lv1=3, Lv5=7

**问题：** 配合 `>= 2` 升级门槛，前期过强（15 秒满级），Boss 机制无法展现。火力 debuff 无意义。

**推荐方案（折中）：**
```csharp
int dmg = 1 + lv;  // Lv1=2, Lv2=3, Lv3=4, Lv4=5, Lv5=6
```
- Lv1 时 DPS 为 8（4 发×2），打 E01(10HP) 需 1.25 秒 — 仍然快速但不过分
- Lv5 时 DPS 为 30（5 发×6），打 Boss(350HP) 需 12 秒 — 足够触发一次阶段切换
- 火力 debuff 在 Lv2 时降为 Lv1（DPS 8→8 无变化），Lv3+ 时有实际影响

#### P2-2: 升级门槛恢复渐进式

**文件：** [PlayerWeapon.cs:99](Assets/Scripts/Player/PlayerWeapon.cs)

**当前值：** `insulinCollected >= 2` — 固定每 2 个升一级

**问题：** 无递增感，前期升级过快。

**推荐方案：**
```csharp
if (insulinCollected >= level)  // 恢复原设计：Lv1→2 需 1 个，Lv2→3 需 2 个...Lv4→5 需 4 个
```
- 配合掉率提升（50%+），第一关可获得约 10-15 个结晶，足够升到 Lv4-Lv5
- 升级有递增难度，保持成长感

#### P2-3: Boss 血量回调

**文件：** [GameDatabase.cs:129-148](Assets/Scripts/Core/GameDatabase.cs)

**当前值：** B01=350, B02=450, B03=650

**问题：** 配合当前伤害，Boss 在 2-3 秒内可被击杀（Lv5 时 DPS 35+），阶段切换机制全部失效。

**推荐方案（在 P2-1 伤害调低后评估是否需要）：**
```csharp
// 如果伤害改为 1+lv，则 Boss 血量可保持当前值或微调：

```
如果保持当前 `2+lv` 伤害不变，建议 Boss 血量翻倍至 700/900/1300。
